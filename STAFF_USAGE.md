# STAFF_USAGE

## 登录

在浏览器打开办公室局域网地址，输入自己的用户名和密码登录。员工不能自己注册账号；没有账号请联系管理员创建账号。

系统会自动用当前登录员工记录操作人。正常登录后不能手动改成其他人的名字。

如果系统要求修改密码，会自动进入 `修改密码` 页面。请输入当前密码、新密码和确认新密码。修改完成后才能继续使用系统。

首次使用新数据库时，系统会进入 `创建第一个管理员` 页面。管理员创建完成后，可以在 `用户管理` 页面创建其他员工账号。

角色说明：

- 管理员：可以管理用户和查看所有内部页面。
- 业务员：可以使用报价、查询、供应商、报关资料等日常业务功能，但不能管理用户。
- 跟单：可以使用报价、查询、供应商、报关资料等日常业务功能，但不能管理用户。

旧的共享访问码默认不可用。只有开发人员在 `.env` 里开启 `ENABLE_LEGACY_ACCESS_CODE=true` 后，才可作为临时应急访问方式使用。临时访问模式不能进入用户管理。

## 用户管理

只有管理员可以进入 `用户管理`。

管理员可以：

- 新增员工账号。
- 修改用户名、显示名称和角色。
- 重置员工密码。
- 启用或禁用员工账号。
- 删除员工账号。

新增、编辑、重置密码、启用/禁用和删除用户时，管理员都需要输入自己的管理员密码确认。

删除账号不会删除该员工之前的操作记录。历史记录会保留操作人姓名，但不再关联到这个用户账号。

离职员工请禁用账号，不要改给新人使用。新员工应该创建新的用户账号。

## 上传完整报价单

1. 点击 `上传完整报价单`。
2. 确认操作员姓名显示当前登录员工。正常登录时该字段不可改。
3. 上传 `.xlsx` 报价文件。
4. 查看导入预览。
5. 如果预览显示 GTS、OEM、工厂或价格变化，请选择保留旧值或使用新值。
6. 如果预览显示单位变化，请确认提醒内容是否正确。
7. 如果匹配产品的非空产品字段不同，勾选需要更新的字段。
8. 如果缺少工厂、单位或价格，系统会禁止导入，请先修改 Excel。
9. 输入自己的登录密码确认。
10. 点击 `确认导入`。

系统会保留旧报价记录，并新增新的历史报价记录。若本次上传的 GTS/OEM、工厂、单位、价格与系统中已有报价完全一致，系统不会新增重复报价行，而是更新已有报价行的确认日期和更新人。MVP 不提供删除功能。

## 生成报价单

1. 点击 `生成报价单` 或首页的 `按 GTS/OEM 生成报价单`。
2. 确认操作员姓名显示当前登录员工。
3. 上传 `.xlsx` 需求清单。
4. 需求清单可以只包含 `GTS`，也可以包含 `GTS`、`OEM`、英文描述、数量、单位、备注等列。
5. 查看生成预览中的匹配、未匹配、冲突和多候选行。
6. 如果一行有多个候选报价，选择要使用的历史报价。
7. 如果不想使用某一行，取消勾选 `使用此行`。
8. 点击 `生成 Excel`。

如果需求文件中所有行都没有 GTS 或 OEM，系统会拒绝生成。若只有部分行缺少 GTS/OEM，系统会保留这些行并显示提醒。

生成文件只用于立即下载，MVP 不会永久保存到 `generated/`。

生成 Excel 时，如果上传的需求清单提供了 OEM、英文描述或单位，系统会保留上传值；如果这些字段为空，系统会使用选中的历史报价数据补齐。

## 采购合同

`采购合同` 用来保存采购合同号、供应商、合同状态和采购明细。它是后续做报关批次、报关明细和报关 Excel 的基础数据。当前阶段只保存数据，不生成正式采购合同 Excel 模板。

规则：

- 一个采购合同只能对应一个供应商。
- 合同号必须唯一。
- 管理员和跟单可以新增、编辑、取消采购合同；业务员只能查看。
- 新增、编辑、取消合同和添加明细时，需要输入自己的登录密码确认。

创建采购合同时需要选择供应商并填写合同号。合同号现在手动填写，自动编号属于后续功能。

合同明细可以选择系统产品，也可以手动填写 GTS、OEM、中文品名、英文描述、数量、单位、人民币单价、毛重、件数、体积和备注。系统会根据 `数量 × 人民币单价` 计算明细金额，并自动汇总合同总金额。

毛重和件数很重要，因为后续报关净重会按公司规则计算：

```text
净重 = 毛重 - 件数
```

当前阶段不会在采购合同里计算或校验净重；后续报关明细预览和导出阶段会检查这些数据。

## 查询数据库

点击 `查询数据库`，可按以下字段查询：

- GTS
- OEM
- 英文描述
- 品名
- 工厂

GTS/OEM 查询支持模糊匹配。例如搜索 `test001` 或 `001`，可以匹配到 `GTSTEST001`。如果存在完全匹配结果，系统只显示完全匹配结果。

查询操作目前不写入操作记录。

## 供应商

点击 `供应商` 可以查看、搜索、新增和编辑供应商资料。

供应商搜索支持供应商全称、简称、别名、城市和产品范围。新增或编辑供应商时，操作人会使用当前登录员工，并需要输入自己的登录密码确认。

供应商资料必须填写全称和唯一简称，别名可选。多个别名请用中文逗号或英文逗号分隔。质量评分、价格评分、配合评分都是手动维护的 1-5 分评分，可以留空。

上传完整报价单时，如果报价行的工厂名称与已有供应商全称、简称或别名唯一匹配，系统会自动把该报价行关联到供应商；如果没有匹配、存在多个匹配，或 Excel 工厂为空，需要在导入预览页的 `供应商匹配` 区域直接处理。所有供应商都处理完成后才能导入。空白工厂必须选择或新建供应商，系统会用确认后的供应商名称补入工厂字段。

供应商评分和联系方式不影响报价导入，可以之后在 `供应商` 列表的更多页面补充。历史报价、生成报价预览和导出的 Excel 会优先显示供应商简称；没有供应商关联时才显示原始工厂文字。

## 报关资料

`报关资料` 用来维护报关主数据，例如 `减震器`、`支架`、`水泵`、`滤清器`。这是后续做报关明细、报关要素和报关 Excel 的基础资料。

HS Code 不再作为单独模块显示，旧的 HS Code 上传/导出入口已经合并到 `报关资料`。目前 `products.hs_code` 仍作为旧流程兼容字段保留；未来方向是产品通过报关映射关联到 `报关资料库` 里的 HS Code。

当前阶段已经支持 `报关资料库` 和 `产品报关映射`，并保留旧 HS Code 上传/导出兼容流程。暂时不生成报关明细、不导出最终报关 Excel。

### 报关资料库

管理员和跟单可以新增、编辑、启用或停用报关资料；业务员只能查看。

新增报关资料时需要填写：

- 报关中文品名
- HS Code
- 第一单位
- 第一单位来源
- 第一单位小数位
- 可选第二单位、第二单位来源和第二单位小数位
- 可选申报要素模板和备注

`unit_1 / unit_2` 是报关使用的法定单位。系统不假设第一单位一定是 `个`，也不假设第二单位一定是 `千克`。

例子：

- 只有一个单位也可以是 `千克`，来源可以是 `毛重`。
- 两个单位可以是 `台 + 千克`，也可以是 `套 + 个`。

`来源` 表示以后生成报关明细时这个单位的数量从哪里来：

- 数量：以后使用产品或合同数量
- 毛重：以后使用毛重
- 净重：以后使用净重
- 体积：以后使用体积
- 件数：以后使用包装件数
- 手动填写：以后在报关明细预览中人工填写

申报要素模板现在只是文本保存，例如：

```text
品名：{customs_name_cn}
用途：汽车用
品牌：无品牌
零部件编号：{part_numbers}
适用车型：{model_for_declaration}
```

### 产品报关映射

`产品报关映射` 用来把系统里的产品 / GTS No. 连接到一套报关资料。

规则：

- 一个 GTS No. 只能对应一套报关资料。
- 一套报关资料可以对应多个 GTS No.。
- 如果同一个产品重新选择报关资料，系统会更新原来的映射，不会保留多条当前映射。
- 管理员和跟单可以新增、编辑映射；业务员只能查看。

新增或编辑映射时，需要选择：

- 产品
- 报关资料
- 可选申报零部件编号
- 可选适用车型
- 可选材质
- 可选品牌
- 可选备注
- 输入自己的登录密码确认

后续报关模块会优先使用 `产品报关映射 -> 报关资料库 HS Code`。旧产品里的 HS Code 只是兼容旧上传/导出流程的备用字段。

### 报关资料缺失检查

`报关资料缺失检查` 用来提前发现后续报关文件生成会卡住的问题。

页面会提示：

- 产品还没有报关映射。
- 已映射报关资料缺少 HS Code。
- 后续单位来源需要毛重，但系统找不到毛重。
- 后续单位来源需要件数，但系统找不到件数。
- 后续单位来源需要净重，但系统无法计算净重。
- 报关资料缺少申报要素模板。
- 单位来源是手动填写，后续做报关明细时需要人工补数量。

毛重来自报价上传中的 `G.W.` / `gross_weight` 字段。

件数来自报价上传中的 `packages` 字段，不是 `item/package`。

公司当前净重规则：

```text
净重 = 毛重 - 件数
```

如果毛重缺失、件数缺失，或 `毛重 - 件数 <= 0`，系统会显示阻止后续导出的错误。其中净重计算错误会显示：

```text
净重计算错误：毛重 - 件数 必须大于 0。
```

### 批量上传

1. 点击 `报关资料` > `批量上传`。
2. 确认操作员姓名显示当前登录员工。
3. 上传 `.xlsx` 文件。
4. 旧兼容文件中必须有 GTS 列和 HS Code 列。
5. GTS 列使用系统现有 GTS 表头别名；HS Code 列支持 `HS`、`hscode`、`海关编码`、`hs code`。
6. 查看预览，输入自己的登录密码后点击 `确认更新`。

如果数据库里同一个 GTS 已经有 HS Code，系统会直接覆盖为本次上传的新 HS Code。当前兼容上传只更新产品旧 HS Code；如果文件缺少报关品名、第一单位和第一单位来源，不会自动创建报关资料库记录。

### 导出报告

1. 点击 `报关资料` > `导出报告`。
2. 确认操作员姓名显示当前登录员工。
3. 上传包含 GTS 列的 `.xlsx` 文件。
4. 查看预览。
5. 点击 `导出 Excel`。

当前导出的兼容 Excel 会保持上传文件中的 GTS 顺序，只包含 `GTS`、`OEM`、`HS Code` 三列。完整报关明细和申报要素导出属于后续阶段。

## 操作记录

经理可以打开 `操作记录` 查看上传报价单、生成报价单、报关资料上传、报关资料导出等记录。
